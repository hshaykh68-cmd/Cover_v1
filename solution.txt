# Complete Build Error Analysis and Solutions

## Error Category 1: Missing Material Icons Extended Library

### Problem:
You're using `androidx.compose.material:material-icons-core` which only includes a limited set of icons. The following icons are not available in the core library:
- `FolderOpen`, `PhotoLibrary`, `Apps`, `Error`, `VisibilityOff`, `SdStorage`, `Verified`
- `Description`, `Photo`, `Audiotrack`, `Download`, `InsertDriveFile`, `PlayCircle`, `Videocam`
- `DeleteSweep`, `Shield`, `NoPhotography`
- `DoNotDisturb`, `Cloud`, `CameraAlt`, `Help`
- `Adb`, `Image`

### Solution:
**Copy and paste this into your `app/build.gradle.kts` or `app/build.gradle`:**

```gradle
dependencies {
    // Replace material-icons-core with material-icons-extended
    // Remove this line if present: implementation("androidx.compose.material:material-icons-core:1.x.x")
    // Add this line:
    implementation("androidx.compose.material:material-icons-extended:1.7.5")
    
    // Or keep both if you want:
    implementation("androidx.compose.material:material:1.7.5")
    implementation("androidx.compose.material:material-icons-extended:1.7.5")
}
```

**Add these imports to affected files (EmptyStates.kt, GalleryScreen.kt, IntruderLogsScreen.kt, etc.):**

```kotlin
import androidx.compose.material.icons.Icons
import androidx.compose.material.icons.filled.*
import androidx.compose.material.icons.outlined.*
import androidx.compose.material.icons.rounded.*
```

## Error Category 3: ListenableFuture Extension Function Issue

### Problem:
The `await()` extension function for `ListenableFuture` is not properly recognized

### Error Line:
```
e: file:///.../IntruderCaptureManager.kt:124:63 Unresolved reference. None of the following candidates is applicable because of receiver type mismatch: 
private final suspend fun <T> ListenableFuture<TypeVariable(T)>.await(): TypeVariable(T) defined in com.cover.app.data.intruder.IntruderCaptureManager
```

### Solution:
**Copy and paste this into your `app/build.gradle.kts` or `app/build.gradle`:**

```gradle
dependencies {
    // Add this dependency for coroutines with Play Services
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-play-services:1.7.3")
}
```

**Copy and paste this at the top of `IntruderCaptureManager.kt`:**

```kotlin
import kotlinx.coroutines.tasks.await
```

**If you have a custom extension function, remove it and use the library version:**

```kotlin
// REMOVE this custom function if it exists:
// private suspend fun <T> ListenableFuture<T>.await(): T { ... }

// USE the library import instead:
import kotlinx.coroutines.tasks.await

// Usage:
val result = someListenableFuture.await()
```

## Error Category 6: Experimental Compose APIs

### Problem:
Using experimental Foundation APIs without proper opt-in

### Error Lines:
```
e: file:///.../MediaViewerScreen.kt:42:22 This foundation API is experimental and is likely to change or be removed in the future.
e: file:///.../MediaViewerScreen.kt:50:39 This foundation API is experimental...
```

### Solution:
**Copy and paste this at the top of `MediaViewerScreen.kt`:**

```kotlin
import androidx.compose.foundation.ExperimentalFoundationApi

@OptIn(ExperimentalFoundationApi::class)
@Composable
fun MediaViewerScreen() {
    // Your existing code
}
```

**Or apply to specific composables:**

```kotlin
@OptIn(ExperimentalFoundationApi::class)
@Composable
fun YourSpecificComposable() {
    // Code using experimental APIs
}
```
